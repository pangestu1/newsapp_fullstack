üéØ Overview

NewsApp adalah RESTful API backend untuk aplikasi berita dengan sistem role-based authentication. API ini mendukung operasi CRUD untuk berita, komentar, dan manajemen user.

Role Permissions
Admin Full access ke semua endpoint
Penulis Buat, edit, hapus berita (milik sendiri) + komentar
Pembaca Baca berita dan buat komentar

üîê Authentication

POST PUBLIC
/auth/register
Mendaftarkan user baru ke sistem.

Headers:

Content-Type: application/json

Body:

{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123",
  "role": "pembaca"
}

Parameters:

Parameter Type Required Description
name string ‚úÖ Nama lengkap user
email string ‚úÖ Email user (unik)
password string ‚úÖ Password (min 6 karakter)
role string ‚ùå admin, penulis, pembaca (default)

Response Success (201):

{
  "message": "User created successfully",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "pembaca"
  }
}
POST PUBLIC
/auth/login
Login user dan mendapatkan JWT token.

Body:

{
  "email": "john@example.com",
  "password": "password123"
}

Response Success (200):

{
  "message": "Login successful",
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "pembaca"
  }
}

üóÑÔ∏è Database Structure

üìä Table: Users

Menyimpan informasi user dan kredensial login.

Column Type Constraints Description
id INT PRIMARY KEY, AUTO_INCREMENT ID unik user
name VARCHAR(255) NOT NULL Nama lengkap user
email VARCHAR(255) NOT NULL, UNIQUE Email user (unik)
password VARCHAR(255) NOT NULL Hash password terenkripsi
role ENUM DEFAULT 'pembaca' admin, penulis, pembaca
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP Waktu pembuatan akun
updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP Waktu update terakhir

üìä Table: News

Menyimpan artikel berita yang dibuat oleh penulis.

Column Type Constraints Description
id INT PRIMARY KEY, AUTO_INCREMENT ID unik berita
title VARCHAR(255) NOT NULL Judul berita
content TEXT NOT NULL Konten lengkap berita
image VARCHAR(255) NULLABLE Path/nama file gambar
author_id INT FOREIGN KEY (users.id) ID penulis berita
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP Waktu pembuatan berita
updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP Waktu update terakhir

üìä Table: Comments

Menyimpan komentar user pada artikel berita.

Column Type Constraints Description
id INT PRIMARY KEY, AUTO_INCREMENT ID unik komentar
content TEXT NOT NULL Isi komentar
news_id INT FOREIGN KEY (news.id) ID berita yang dikomentari
user_id INT FOREIGN KEY (users.id) ID user yang komentar
created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP Waktu pembuatan komentar
updated_at TIMESTAMP ON UPDATE CURRENT_TIMESTAMP Waktu update terakhir

üöÄ Quick Start Examples

1. Register User

curl -X POST http://localhost:5000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Alice Johnson",
    "email": "alice@example.com",
    "password": "alice123",
    "role": "penulis"
  }'

2. Login

curl -X POST http://localhost:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "alice@example.com",
    "password": "alice123"
  }'

3. Create News (dengan image)

curl -X POST http://localhost:5000/api/news \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "title=Berita Terbaru" \
  -F "content=Konten berita..." \
  -F "image=@/path/to/image.jpg"

4. Get All News

curl "http://localhost:5000/api/news?page=1&limit=5&search=teknologi"

5. Get News Detail

curl "http://localhost:5000/api/news/1"

6. Create Comment

curl -X POST http://localhost:5000/api/comments \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Berita yang sangat informatif!",
    "news_id": 1
  }'

7. Get Comments for News

curl "http://localhost:5000/api/comments/news/1"

8. Get All Users (admin only)

curl "http://localhost:5000/api/users" \
  -H "Authorization: Bearer ADMIN_TOKEN"

9. Update User Role (admin only)

curl -X PUT http://localhost:5000/api/users/2/role \
  -H "Authorization: Bearer ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "role": "penulis"
  }'

2. Create News (with image)

curl -X POST http://localhost:5000/api/news \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "title=Berita Terbaru" \
  -F "content=Konten berita..." \
  -F "image=@/path/to/image.jpg"

3. Get News with Search

curl "http://localhost:5000/api/news?page=1&limit=5&search=teknologi"
